Domain: 8 recipes stored in SQLite → cached in Redis under namespace ac29.

Fetch single recipe via @cached + MsgPack

First call: body of _cached_fetch_recipe() executes, SQLite queried (140 ms simulated), result serialised with MsgPack and stored in Redis. Subsequent calls: aiocache reads from Redis, deserialises with MsgPack (~1 ms). DB never touched.

# Named cache — configured once at module import
caches.set_config({
    "ac29_recipes": {
        "cache": "aiocache.RedisCache",
        "namespace": "ac29",
        "ttl": 180,
        "serializer": {"class": "aiocache.serializers.MsgPackSerializer"},
        "plugins": [
            {"class": "aiocache.plugins.HitMissRatioPlugin"},
            {"class": "aiocache.plugins.TimingPlugin"},
        ],
    }
})

@aiocache_cached(alias="ac29_recipes", key_builder=_recipe_key_builder)
async def _cached_fetch_recipe(recipe_id: int):
    """Body runs ONLY on cache miss."""
    return await _db_fetch_recipe(recipe_id)

Batch fetch via cache.multi_get() / cache.multi_set()

multi_get(keys) issues a single Redis MGET for all IDs simultaneously — one network round-trip regardless of count. Cache misses are fetched from DB in parallel with asyncio.gather(), then stored back via a single multi_set().

be/routers/s29_aiocache.py — multi_get / multi_set 
cache = caches.get("ac29_recipes")
keys = [str(i) for i in id_list]

# ONE Redis MGET — latency constant regardless of key count
cached_values = await cache.multi_get(keys)

# misses → concurrent DB fetch
fetched = await asyncio.gather(*[
    _db_fetch_recipe(rid) for rid in miss_ids
])

# ONE Redis MSET for all DB results
await cache.multi_set(pairs, ttl=180)

cache.delete(key)

Manually evict one recipe key from the ac29 namespace. The next GET for that ID will be a cache miss.

cache.clear()

Evict every key in the ac29 namespace atomically. Equivalent to KEYS ac29:* + DEL. All subsequent GETs will be misses.

be/routers/s29_aiocache.py — manual operations 
# Check existence (no value fetched)
pre_exists = await cache.exists(str(recipe_id))

# Delete one key (namespace applied automatically)
deleted = await cache.delete(str(recipe_id))

# Evict entire namespace — uses configured namespace="ac29"
await cache.clear()

Why MsgPack over JSON?

JSON (text)
{"id": 1, "name": "Pizza"}
→ stored as UTF-8 text
→ Python dict → json.dumps() → bytes
→ slow for nested structs
→ larger payload (quoted keys + values)
MsgPack (binary)
81 a2 69 64 01 a4 6e 61 6d 65 …
→ compact binary format
→ Python dict → msgpack.packb() → bytes
→ 2–3× faster serialisation
→ 30–50% smaller in Redis memory

Install: pip install msgpack — aiocache loads it automatically when MsgPackSerializer is configured.

How aiocache plugins work

# HitMissRatioPlugin hooks post_get() on the cache instance:
class HitMissRatioPlugin(BasePlugin):
    async def post_get(self, client, key, ret, ...):
        if ret is not None:
            client.hit_miss_ratio["hits"] += 1   # stored on the cache instance
        else:
            client.hit_miss_ratio["misses"] += 1

# Access after operations:
cache = caches.get("ac29_recipes")
ratio = cache.hit_miss_ratio   # {"hits": 42, "misses": 7, "hit_ratio": 0.857}

TimingPlugin hooks pre_get / post_get to record per-operation latency — transparent to application code. Plugins are registered once in caches.set_config() and apply automatically to every operation on the named cache.
