How each demo works

⬤ CACHE MISS

First fetch: simulates a slow database read (120–200 ms). Result is then stored in cache. Next call will be a HIT.

⬤ CACHE HIT

Data returned directly from RAM or Redis in < 2 ms. The database is never touched. Observe the latency drop.

✕ CLEAR CACHE

Evicts all entries. The next fetch will be a MISS again — let you observe the full miss → populate → hit cycle.
