What is In-Memory Caching?

Data is stored directly in the Python process's RAM — no network round-trip, no serialisation, just a dictionary lookup. Access is nanosecond-fast. The trade-off: the cache lives only as long as the process does, and is not shared between multiple workers or app instances.

Scope
Single OS process. Not visible to other workers.
Survival
Cache is wiped on every app restart / deploy.
Best for
Immutable reference data. Country configs, feature flags, compiled regexes.
functools.lru_cache No TTL · maxsize=128

Decorator that wraps any pure function. The return value is memoised forever (or until the process restarts / the cache is cleared). Evicts the Least Recently Used entry when maxsize is hit.

@lru_cache(maxsize=128)
def fetch_country(code: str) -> dict:
# first call: 120 ms DB round-trip
time.sleep(0.120)
return _COUNTRY_DB[code]

# second call: 0 ms — returns from RAM
cachetools.TTLCache TTL: 30 s · maxsize=100

Third-party cache with an automatic Time-To-Live. Entries expire 30 seconds after insertion — ideal for permission checks, feature flags, or any data that changes occasionally. Protected by threading.Lock() for thread safety.

_store = TTLCache(maxsize=100, ttl=30)
_lock = threading.Lock()

with _lock:
if key in _store:
return _store[key], True # HIT
perms = db_lookup(user_id) # MISS
_store[key] = perms

💡 Fetch the same user twice — first is a MISS (~150 ms), second is a HIT (<1 ms). Wait 30 seconds and the entry auto-expires — next fetch is a MISS again.
