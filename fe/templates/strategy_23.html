What is Cache-Aside (Lazy Loading)?

The application code itself is responsible for loading data into the cache. On a read: check Redis first. On a miss, query the database (real SQLAlchemy ORM — not a fake dict), store the result in Redis, then return it. Data enters the cache lazily — only when it is first requested.

This demo adds one important upgrade over the basic Redis pattern: negative caching. When an ID does not exist in the database, a null sentinel is stored in Redis with a short 15-second TTL. Repetitive 404 lookups then never reach the database at all.

Positive TTL
120 seconds. Real orders cached after first DB read.
Negative TTL
15 seconds. null stored for non-existent IDs. Short TTL = heals quickly.
Data source
Real SQLAlchemy ORM session.get(Order, id) on every miss.

Complete Flow Diagram

Cache key: order:{id}  |  DB table: orders via SQLAlchemy async ORM

GET order:{id}
1.
HIT (positive) — Redis returns JSON blob
→ latency ~1 ms  |  DB not touched  |  TTL decrement shown
2.
HIT (negative) — Redis returns "null" sentinel
→ 404 returned immediately  |  DB NOT touched  |  latency ~1 ms
3a.
MISS + order exists — Redis has no entry
→ session.get(Order, id) via SQLAlchemy  ~180 ms
→ SETEX order:{id} 120 <json>  (stored in Redis)
→ return order data
3b.
MISS + order NOT found — Redis has no entry
→ session.get(Order, id) returns None  ~180 ms
→ SETEX order:{id} 15 "null"  (negative sentinel, short TTL)
→ return 404  |  next call hits Redis, NOT DB (step 2)
Try these steps:
① Fetch order 1 → MISS (~180 ms, SQLAlchemy query)
② Fetch order 1 again → HIT (<2 ms, Redis)
③ Fetch order 99 → MISS then → 404 (negative cache written)
④ Fetch order 99 again → HIT + 404 (<2 ms, DB NOT touched)
⑤ Clear → all entries evicted → repeat cycle
Actual backend code (simplified)
# check cache
raw = await redis.get(f"order:{id}")

if raw is not None:
val = json.loads(raw)
if val is None:
raise 404 # negative hit
return val # positive hit

# miss — hit the DB
order = await db.get(Order, id)
if order:
await redis.setex(key, 120, dump(order))
return order
else:
await redis.setex(key, 15, "null")
raise 404
